# UI_VISION

## Cel wizualny
UI ma być spokojne, nowoczesne i wiarygodne. Ma wyglądać jak prawdziwy produkt pomagający człowiekowi, a nie jak losowo wygenerowany „AI wellness landing”.

## Charakter aplikacji
- minimalizm,
- dużo oddechu,
- czytelna hierarchia,
- proste komunikaty,
- brak chaosu wizualnego,
- brak zbędnych efektów.

## Emocja, jaką ma dawać produkt
- bezpieczeństwo,
- spokój,
- nadzieja,
- porządek,
- zaufanie.

## Styl komunikacji w UI
- krótko,
- konkretnie,
- wspierająco,
- bez patosu,
- bez marketingowego nadęcia,
- bez generycznych tekstów typu „discover your journey”.

## Ogólne zasady projektu
- mobile-first,
- ekran ma prowadzić użytkownika do jednej konkretnej akcji,
- formularze mają być krótkie i lekkie,
- treść ma być łatwa do skanowania,
- priorytetem jest użyteczność, nie dekoracja.

## Relacja do innych dokumentów

## Kierunek estetyczny
- biała baza,
- akcent zielony i niebieski zgodnie z design systemem,
- lekkie powierzchnie,
- miękkie separatory,
- prosty, nowoczesny font sans-serif,
- wyraźne CTA,
- spokojny, profesjonalny odbiór.

## Zasada nadrzędna
Każdy ekran ma wyglądać jak część jednego sensownego produktu, a nie jak osobny eksperyment wizualny.
