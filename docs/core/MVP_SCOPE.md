# MVP_SCOPE

## Cel MVP
Zbudować minimalną wersję aplikacji, która spełnia wymagania projektu i pokazuje pełny działający przepływ użytkownika.

## W zakresie MVP

### Publiczne
- strona główna,
- ściana inspiracji,
- generator inspiracji,
- formularz anonimowej skrzynki,
- sekcja pomocy,
- rejestracja,
- logowanie,
- wylogowanie.

### Po zalogowaniu
- panel użytkownika,
- lista własnych wpisów,
- dodawanie własnych wpisów,
- edycja własnych wpisów.

### Dla administratora
- panel admina,
- podgląd wszystkich treści,
- edycja wszystkich treści,
- zmiana statusów,
- zarządzanie sekcją pomocy,
- zarządzanie inspiracjami.

### Technicznie
- Express,
- baza danych,
- formularze HTML,
- POST do zapisu danych,
- seed danych testowych,
- salted hash hasła,
- role użytkownika i admina.

## Poza zakresem MVP
- powiadomienia,
- komentarze,
- polubienia,
- system znajomych,
- wiadomości między użytkownikami,
- rozbudowana moderacja,
- PWA,
- aplikacja mobilna,
- płatności,
- rozbudowane profile,
- zbędne animacje i efekty wizualne.

## Priorytet budowy
1. działające logowanie i role,
2. zapisywanie danych w bazie,
3. inspiracje,
4. anonimowa skrzynka,
5. sekcja pomocy,
6. panel admina,
7. dopiero potem drobne ulepszenia.

## Zasada
Jeśli coś nie wspiera zaliczenia projektu albo głównego celu aplikacji, nie powinno wejść do MVP.
